let health = 10;
while (health > 0) {
  console.log("Health:", health);
  health -= 3; // ⚠️ MUST change condition or: infinite loop!
}

// DO...WHILE — runs at least once, THEN checks
let answer;
do {
  answer = prompt("Type yes:"); // always shows once
} while (answer !== "yes");

₿ The Float Danger — Full Breakdown
Binary can't perfectly represent 0.1. In binary, 0.1 is 0.0001100110011... repeating forever — like 1/3 in decimal (0.333...). The computer truncates it. Those tiny errors add up. In finance, they become real money.
❌ The Dangerous Way
⚠️ Using floats for money
// User buys ₿0.1 + ₿0.2 of Bitcoin
let buy1 = 0.1; // ₿
let buy2 = 0.2; // ₿
let total = buy1 + buy2;
console.log(total);
// 0.30000000000000004 ← WRONG

// At ₹35,00,000 per BTC:
// Error = 0.00000000004 BTC
// = ₹0.00014 per txn
// × 1 crore txns = ₹1,40,000 LOST
Every transaction has a tiny error. At scale — millions of transactions — those tiny errors become massive losses or gains. Real exchanges have been sued over this.
✅ The Safe Way — Integer Arithmetic
✅ Work in smallest unit (satoshis)
// 1 Bitcoin = 100,000,000 satoshis
// NEVER store BTC as 0.1
// Store as SATOSHIS (integers)

const SAT = 100_000_000; // 1 BTC
let buy1 = 10_000_000; // 0.1 BTC
let buy2 = 20_000_000; // 0.2 BTC
let total = buy1 + buy2;
// 30000000 ← EXACT. Always.

// Display only: divide at the end
console.log(total / SAT + " BTC");
// "0.3 BTC" ← clean
Bitcoin does this natively. Ethereum uses "wei" (10⁻¹⁸ ETH). Banks store in paise, not rupees. The rule: compute in integers, display in decimals.
3 Solutions — Choose Based on Context
1. Integer Arithmetic (Finance & Crypto)
// Store in paise / satoshis / wei / cents
// NEVER do math on the decimal form
let price = 19999; // ₹199.99 in paise
2. toFixed() — For Display Only
// Only for showing the user a clean number
// toFixed() returns a STRING — don't math on it!
(0.1 + 0.2).toFixed(2) // "0.30" ← string, display only
3. Number.EPSILON — For Comparisons
// When you MUST compare floats
function nearlyEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}
nearlyEqual(0.1+0.2, 0.3); // true ✓

📓 Chapter 4 — Summary
📖 Arjun's Notes + Anjali's additions ✏️
🧭
if / else if / else — checks conditions top-to-bottom. First true one runs, everything else is skipped. Order matters completely.
🔁
for = known count. while = unknown count, checks before. do...while = always runs once, then checks. Always update the condition variable or: infinite loop.
🔢
One number type for everything. IEEE 754 double-precision. Math object: round, floor, ceil, max, min, abs, pow, sqrt, PI, random. NaN and Infinity are both typeof "number".
₿
0.1 + 0.2 ≠ 0.3 in binary. For money/crypto: store in integers (paise, satoshis). Display with toFixed(). Compare floats with Number.EPSILON. Real exchanges have lost real money from this.
🎲
Random integers: Math.floor(Math.random() × (max−min+1)) + min. Step 1: scale. Step 2: floor. Step 3: shift. That formula is the only right way.
🔤
Strings are immutable — methods return NEW strings. Key methods: toUpperCase, trim, slice, includes, replace, replaceAll, split, indexOf. Template literals beat concatenation always.
🎓
Anjali's lesson: Explaining something to someone else is the fastest way to find out what you didn't actually understand. Teach to learn.
