The 3 Reasons C++ Couldn't Be in Browsers
Why C++ Was Impossible
☠️ Security
C++ can read your files, open sockets, crash the OS. A website running raw C++ could install malware or wipe your drive.
💾 Memory (1995)
Average PC had 4–8 MB RAM. Running a sandboxed C++ runtime would have left zero room for Windows 95 AND a browser.
🚫 Too Hard
Web designers, not kernel engineers, needed to write "glue" code. C++ with manual memory management was completely wrong for that audience.
✅ JS Fixed All Three
Sandboxed. Tiny footprint. Simple syntax. Automatic garbage collection. Designed specifically for the browser environment.

BONUS How a Compiler Actually Works 📍 Under the Hood
Compiler Theory — Simplified
Source Code → Output: The 5 Stages
1. Lexer
Breaks raw text into tokens.
console . log ( "hi" )
2. Parser
Tokens → AST tree of meaning. Checks grammar rules.
3. Semantic
Are the types right? Does this variable exist? Type checking.
4. Code Gen
AST → bytecode or machine code instructions.
5. Execute
CPU runs instructions. Output appears.
Build Your Own JS Compiler INTERACTIVE
Here's a tiny compiler written in C++ that handles console.log("string") and console.log(a+b) — exactly what V8 does, just stripped to the bone. Toggle between the real C++ source and a simulated run.
mini_v8.cpp — annotated
// STAGE 1 — LEXER (tokenizer)
// Find the ( and ) — extract what's inside
int open = code.find('(');
int close = code.rfind(')');
string inside = code.substr(open+1, close-open-1);

// STAGE 2 — PARSER + SEMANTIC
// First char is " → it's a string literal
if(inside[0] == '"'){
  // Strip the quotes and print
  cout << inside.substr(1, inside.size()-2);
}
else{
  // STAGE 3 — EVALUATE expression
  // Walk chars to parse left operand
  int index=0, a=0, b=0;
  while(inside[index]!='+' &&
       inside[index]!='-' &&
       inside[index]!='*' &&
       inside[index]!='/'){
    a = a*10 + (inside[index]-'0');
    index++;
  }
  char op = inside[index++]; // operator
  // Walk remaining chars for right operand
  while(index < inside.size()){
    b = b*10 + (inside[index]-'0');
    index++;
  }
  // STAGE 4 — CODE EXECUTION
  if(op=='+') cout << a+b;
  else if(op=='-') cout << a-b;
  else if(op=='*') cout << a*b;
  else cout << a/b;
}

What Real V8 Adds On Top
This snippet handles
String literals. Integer arithmetic. One function call. ~60 lines of C++.
Real V8 adds
Garbage collection. JIT optimization. Closures. Prototype chains. Async/await. ~2M lines of C++.
The key insight
Both are the same idea: read text → parse meaning → execute. Scale is the only difference.
