SCENE 04 Variables — Named Boxes 📍 The Concept Zone
A variable = a named box in RAM
NAME
"Arjun"
addr: 0x1A2F
AGE
20
addr: 0x1A30
IS_STUDENT
true
addr: 0x1A31
THE THREE KEYWORDS
// var — old school, leaks everywhere
if (true) {
  var leak = "I escaped the block!";
}
console.log(leak); // "I escaped the block!" 😬

// let — block scoped, reassignable
let score = 0;
score = 100; // ✓ allowed

// const — block scoped, reference locked
const NAME = "Arjun";
// NAME = "Rahul"; ❌ TypeError
Think of it: var doesn't respect walls. let respects walls but lets you change what's inside. const locks the label tag itself.
Quick Reference
var vs let vs const
Keyword Scope Reassign Redeclare Hoisted as Use when
var Function 🚨 ✓ Yes ✓ Yes undefined Avoid it.
let Block ✓ ✓ Yes ✗ No TDZ ⚡ Counters, loops, anything that changes.
const Block ✓ ✗ No ✗ No TDZ ⚡ Default choice. Everything that shouldn't change.
TDZ = Temporal Dead Zone — the variable exists but accessing it before the declaration line throws a ReferenceError.
SCENE 05 Data Types — 8 Kinds of Box 📍 The Memory Room
The computer stores everything as bytes.
Types tell it what those bytes MEAN.
🔤 string
Text. Any characters between quotes.
let name = "Arjun" ; 'hello' · `hi ${name}`
🔢 number
Integers AND decimals. No float/int split.
42 · 3.14 · NaN · Infinity
✅ boolean
Only two values. True or false. That's it.
true · false
👻 undefined
Declared, but nothing assigned. JS's "empty by default."
let x; // undefined
🚫 null
Intentionally empty. You set this. "Nothing here on purpose."
let data = null;
🔮 symbol
Unique identifier. Never equal to anything else. Even itself.
Symbol('id')
🦕 bigint
Numbers too large for number type. Add n suffix.
9007199254740991n
📦 object
Key-value pairs. Arrays, functions — all objects under the hood.
{ name: "Arjun" }

SCENE 06 Value vs Reference — The Big Brain Moment 📍 Memory Land
Primitives copy the VALUE.
Objects copy the ADDRESS.
ARJUN
So when I do let b = a — it depends on what a is?
CONCEPT
If a is a number or string: b gets a fresh copy. They're independent.
If a is an object: b gets the same address. Touch b, you touch a.
Value vs Reference
Step through both examples — watch what changes in memory.
By Value (Primitives)
let a = 10;
let b = a; // copy
b = 20;
Memory
0x1A00
a
10
0x1A04
b
10
Both a and b start with the same value 10. They're stored at different addresses.

📖 Arjun's Notes — Morning After
🧱
HTML = structure. CSS = style. JS = behaviour. A house without electricity is just a pretty shell.
📦
Variables are named boxes in RAM. Use const by default. let when you need to reassign. Never use var — it leaks out of blocks.
🏷️
7 primitives: string, number, boolean, undefined, null, symbol, bigint. 1 non-primitive: object (includes arrays and functions).
🐛
typeof null → "object" is a 30-year-old bug. null is a primitive. Always check value === null directly.
🧠
Primitives copy by value — independent. Objects copy by reference — same address, shared data. Change one, both feel it.
⚖️
